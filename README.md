# 電影訂票與評論系統

## 專案目的

本專案以電影院購票與評論整合系統為主題。
觀察目前市場上的電影相關網站，購票平台（如威秀影城）與評論平台（如 IMDb、Letterboxd）長期各自獨立。購票網站不呈現觀眾評價，評論網站也不提供購票入口。但觀影體驗本身即包含院線環境的一部分，電影院的畫質、音響品質都可能影響觀眾對一部電影的整體感受與評價。
因此本專案嘗試將兩者整合，讓使用者能在同一個平台上瀏覽電影資訊、選擇場次與座位完成訂票，並在觀影後留下評分與文字筆記，串聯購票前後的完整體驗。

## 使用技術

React Router：頁面路由管理、Nested Routes
useState：管理互動狀態、座位選擇、星星評分、表單輸入
useEffect：處理setTimeout載入效果、監聽localStorage token變化
localStorage：儲存登入token，重新整理後維持登入狀態
useParams：從網址取得電影id，載入對應電影資料
useNavigate：跳轉頁面->訂票完成後跳至成功頁
useLocation：接收跨頁面傳遞的訂單資料、記錄登入前的來源頁面
NavLink：導覽列連結，標示使用者目前所在頁面
Outlet：Nested Routes的子頁面渲染位置
RequireAuth元件：路由保護，未登入時會自動導向登入頁
MovieCard元件：重複使用在首頁與電影列表頁
Array.map()：將電影陣列資料渲染成畫面上的卡片列表

## 功能介紹

### 首頁
- 顯示熱門電影列表
- 點擊電影圖片或標題進入不同的電影介紹頁
- 點擊「前往購票」按鈕進入不同的電影購票頁
- 點擊「撰寫評論」按鈕進入不同的電影評論頁

### 電影頁
- 顯示所有電影，依月份分區呈現

### 登入／登出
- 未登入時進入電影介紹、訂票或評論頁面，會自動導向登入頁
- 登入成功後自動跳回原本要前往的頁面

### 每個電影的巢狀路由

#### 電影介紹
- 顯示劇情、演員、片長、分級等資訊

#### 前往訂票
- 選擇日期 → 場次 → 座位
- 依電影顯示不同日期與場次
- 已售座位以灰色顯示且無法點選
- 三者皆選擇完成後會顯示訂單摘要在畫面底部
- 確認訂票後跳轉至訂票成功頁面

#### 撰寫評論
- 選擇觀看日期、評分星等、輸入筆記
- 如果未選擇日期、評分星等會顯示警示
- 點選清除按鈕原本填寫的資訊會消失
- 點選儲存按鈕後會顯示載入中三秒，最後顯示儲存成功，並清除原本的紀錄
- 儲存的資訊會顯示在評論紀錄，頁面刷新後就會不見
